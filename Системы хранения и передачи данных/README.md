# Домашнее задание к занятию "`Кеширование Redis/memcached`" - `Гаврилова Валерия`

### Задание 1

1. Проблемы производительности и скорости
- Высокая задержка (latency) при доступе к данным.
Пример: Прямые запросы к медленным источникам (база данных на диске, удалённые API, вычисления). Кеш в оперативной памяти (RAM) сокращает время доступа с миллисекунд до микросекунд.
- Длительные вычисления.
Пример: Генерация сложных отчётов, рендеринг графики, результаты тяжёлых алгоритмов. Кеширование готового результата избавляет от повторных вычислений.
- «Горячие» данные (часто запрашиваемые).
Пример: Профиль популярного пользователя, главная страница сайта, топ-товары в интернет-магазине. Кеш снимает повторяющуюся нагрузку с основного источника.

2. Проблемы масштабируемости и нагрузки
- Ограниченная пропускная способность основного источника.
Пример: База данных не справляется с пиковым числом запросов в час распродажи. Кеш принимает на себя львиную долю чтений, защищая БД.
- Рост числа пользователей/запросов.
Пример: Приложение становится популярным, и прямой расчёт каждого запроса нагружает сервер. Кеш позволяет обслуживать больше пользователей на том же железе.
- Проблемы «тёплого» запуска.
Пример: После перезапуска приложения все данные нужно заново загружать из БД, что вызывает высокую нагрузку и медленный отклик. Предварительно заполненный кеш (warm cache) решает эту проблему.

3. Проблемы доступности и отказоустойчивости
- Частичные сбои внешних сервисов или БД.
Пример: Внешний API для курсов валют временно недоступен. Кеш с устаревшими, но приемлемыми данными (стратегия Stale-While-Revalidate) позволяет приложению продолжать работу.
- Повышение доступности при сетевых проблемах.
Пример: Кеширование контента на краю сети (CDN) позволяет отдавать статику пользователям, даже если главный сервер временно недоступен.

4. Проблемы экономии ресурсов и стоимости
- Высокая стоимость операций.
Пример: Платные внешние API (например, геокодирование, проверка подлинности), где каждый вызов стоит денег. Кеширование ответов снижает количество вызовов и расходы.
- Экономия вычислительных ресурсов (CPU).
Пример: Видеохостинг, где конвертация видео в разные форматы требует огромных мощностей. Кеширование готовых файлов устраняет необходимость повторной конвертации.
- Сокращение сетевого трафика.
Пример: Кеширование статических ресурсов (CSS, JS, изображения) в браузерах пользователей или на прокси-серверах ISP значительно снижает нагрузку на каналы и серверы.

5. Проблемы согласованности и предсказуемости
- «Всплески» нагрузки, которые сложно предсказать.
Пример: Упоминание в соцсетях приводит к резкому росту трафика на одну страницу (слэш-эффект). Кеш сглаживает пик, равномерно распределяя нагрузку.
- Улучшение пользовательского опыта (UX).
Пример: Кеширование на клиенте (localStorage, Service Worker) позволяет мгновенно отображать интерфейс при повторном посещении, создавая ощущение мгновенной скорости.
